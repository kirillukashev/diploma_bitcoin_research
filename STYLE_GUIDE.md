# Style Guide: Оформление ВКР

Источник: методические рекомендации РАНХиГС ИЭМИТ, направление 38.03.01 «Экономика», 2023.  
ГОСТ-основа: ГОСТ Р 7.0.11, ГОСТ Р 7.0.5, ГОСТ 7.1, ГОСТ 2.105.

---

## 1. Структура работы

Обязательные элементы (в этом порядке):

1. Титульный лист
2. Оглавление
3. Введение
4. Основная часть (2–3 главы, редко 4)
5. Заключение
6. Список литературы

Необязательные элементы (после Заключения, перед Списком литературы):

- Список сокращений и условных обозначений
- Список терминов
- Список иллюстративного материала
- Приложения

**Ориентир по объёму:**

| Раздел | Объём |
| --- | --- |
| Введение | 1–2 страницы |
| Основная часть | 30–50 страниц |
| Заключение | до 1 страницы |

---

## 2. Параметры страницы

| Параметр | Значение |
| --- | --- |
| Формат | A4 (210 × 297 мм) |
| Межстрочный интервал | 1,5 |
| Шрифт основного текста | Times New Roman, **12 pt** |
| Шрифт заголовков глав | Times New Roman, **14 pt** |
| Математические шрифты | Cambria Math, Asana Math |
| Поле левое | 35 мм |
| Поле правое | 10 мм |
| Поля верхнее и нижнее | 15 мм |
| Выравнивание текста | По ширине |
| Нумерация страниц | По центру вверху; первая страница (титул) — без номера |

## 3. Заголовки

### Уровни

| Уровень | Что относится | Нумерация | Шрифт |
| --- | --- | --- | --- |
| H1 — Глава | Введение, главы, заключение | `1`, `2`, `3` | 14 pt, жирный |
| H2 — Параграф | Разделы внутри главы | `1.1`, `1.2` | 12 pt, жирный |
| H3 — Пункт | Подразделы внутри параграфа | `1.1.1`, `1.1.2` | 12 pt, жирный |

H4 и ниже — **не допускаются**.

### Правила оформления

- Слово «Глава» **не пишется** — только номер и название: `1 Теоретический обзор`.
- Каждая глава начинается с **новой страницы**.
- В конце заголовка **точка не ставится**.
- Переносы слов в заголовке **недопустимы**.
- Заголовки выравниваются **по левому краю**, без абзацного отступа.
- Ненумерованные заголовки («Введение», «Заключение», «Оглавление») — по **центру**.
- Отступ от текста выше и ниже заголовка — **3 интервала** (два полуторных).

### В Markdown

```markdown
# 1 Теоретический обзор                    ← H1-глава (ненумерованные тоже # )
## 1.1 Cryptocurrency как класс активов    ← H2-параграф
### 1.1.1 Определение и свойства           ← H3-пункт
```

Итого в Markdown: `#` = глава, `##` = параграф, `###` = пункт. Глубже не идём.

---

## 4. Стиль текста

- **Академический** стиль: без риторических вопросов, многоточий, лирических отступлений.
- Речь **от третьего лица**: не «я получил результаты», а «были получены результаты» / «автором были получены результаты».
- **Настоящее время** — для описания текущего состояния области и цитирования работ других авторов.
- **Прошедшее время** — для собственных результатов: «предложенная модель позволила...».
- Каждая глава **заканчивается** отдельным параграфом «Выводы».

---

## 5. Введение

Обязательные элементы введения:

1. Актуальность темы исследования
2. Степень разработанности темы
3. Цель и задачи работы
4. Теоретическая и практическая значимость (если применимо)
5. Методология и методы исследования
6. Положения, выносимые на защиту
7. Степень достоверности и апробация результатов (при наличии статистических исследований)

---

## 6. Списки

### Маркированный список

Маркер — **тире (–)**, не дефис и не «bullet» (`•`).

```
– первый пункт;
– второй пункт;
– последний пункт.
```

### Нумерованный список (многопредложный пункт)

```
1. Первый пункт. Здесь может быть несколько предложений.
   Продолжение того же пункта выравнивается под первой буквой.

2. Второй пункт. Начинается с заглавной буквы, оканчивается точкой.
```

### Нумерованный список (однопредложный пункт)

```
1) первый пункт;
2) второй пункт;
3) последний пункт.
```

### Двухуровневый список

Подуровни обозначаются **строчными буквами русского алфавита** (кроме ё, з, й, о, ч, ъ, ы, ь) со скобкой: `а)`, `б)`, `в)`, ...

---

## 7. Рисунки

- Нумерация **в пределах главы**: «Рисунок 2.3» = глава 2, третий рисунок.
- Подпись **после** рисунка, по центру: `Рисунок 2.3 — Динамика доходности Bitcoin`.
- Слова «график», «диаграмма» в подписи **не используются**.
- В тексте обязательна **ссылка** на каждый рисунок: «...в соответствии с рисунком 1.2...» или «...(см. рисунок 2.1)...».
- Рисунок и подпись — по **центру страницы**.

### В Markdown

```markdown
![Описание](../data/processed/reg_coef_external.png)

*Рисунок 2.3 — Динамика доходности Bitcoin*
```

---

## 8. Таблицы

- Нумерация **в пределах главы**: «Таблица 2.1».
- Название **над** таблицей, по центру: `Таблица 2.1 — Основные характеристики выборки`.
- В тексте обязательна **ссылка**: «...данные приведены в таблице 2.1...».
- При переносе таблицы на следующую страницу: «Продолжение таблицы 2.1» / «Окончание таблицы 2.1».
- Шрифт внутри таблицы может быть уменьшен, но **не менее 10 pt**.

### В Markdown

```markdown
*Таблица 2.1 — Основные результаты OLS-оценки*

| Модель | R² | Adj. R² |
| --- | ---: | ---: |
| М1: crypto-specific | 0.0092 | 0.0021 |
| М2: external | 0.0441 | 0.0373 |
| М3: все факторы | 0.0476 | 0.0338 |
```

## 9. Формулы

- Нумерация **в пределах главы**: `(2.1)` = глава 2, первая формула.
- Номер — в **круглых скобках**, справа на уровне формулы.
- Пояснения символов — непосредственно под формулой, начиная со слова **«где»** (без двоеточия, без абзацного отступа):

```
E = dQ/dI · (I/Q),  (2.1)

где E — эластичность, Q — количество, I — доход.
```

- Переменные — *курсивом*, функции (sin, ln, exp) — прямым шрифтом.
- Ссылка в тексте: «...в формуле (1.2)...» или «...в формулах (1.5)–(1.8)...».

### В Markdown (LaTeX/KaTeX)

```markdown
$$
E = \frac{dQ}{dI} \cdot \frac{I}{Q} \tag{2.1}
$$

где $E$ — эластичность, $Q$ — количество, $I$ — доход.
```

## 10. Библиографические ссылки

Допускается **только один тип** ссылок во всей работе. Рекомендуется:

**Внутритекстовые** — в круглых скобках:

```
(Иванов И.В. Название работы. М.: Издательство, 2021. С. 45)
```

или

**Подстрочные (сноски)** — нумеруются **сквозно** по всей работе:

```
¹ Иванов И.В. Название работы. М.: Издательство, 2021. С. 45.
```

- Все источники из ссылок **должны быть** в Списке литературы.
- Заимствования без ссылки = плагиат → снятие с защиты.

---

## 11. Список литературы

- Оформляется по **ГОСТ 7.1**.
- Порядок: **алфавитный** (рекомендуется); первый блок — русскоязычные, второй — иностранные.
- Каждая запись нумеруется арабской цифрой с точкой.

### Примеры записей

**Книга:**
```
1. Иванов А.Б. Название монографии. — М.: Издательство, 2020. — 300 с.
```

**Статья в журнале:**
```
2. Кузнецов В.П. Заголовок статьи / В.П. Кузнецов // Журнал. — 2022. — № 3. — С. 15–20.
```

**Электронный ресурс:**
```
3. Название ресурса. — URL: https://example.com (дата обращения: 10.05.2026).
```

**Статья на английском (DOI):**
```
4. Liu Y., Tsyvinski A. Risks and Returns of Cryptocurrency // The Review of Financial Studies. — 2021. — Vol. 34, No. 6. — P. 2689–2727. DOI: 10.1093/rfs/hhaa113
```

## 12. Приложения

- Обозначаются заглавными буквами: **Приложение А**, **Приложение Б**, ... (без букв Ё, З, Й, Щ, Ч, Ъ, Ы, Ь).
- Нумерация рисунков и формул внутри приложения: «Рисунок А.3», формула `(В.1)`.
- На каждое приложение — ссылка в тексте; располагаются в порядке ссылок.
- Перечисляются в оглавлении с номерами, заголовками и страницами.

```
│   └── processed/                     # Финальный датасет и графики для ВКР

│
├── docs/                              # Справочные материалы (публичные)

│
├── .local/                            # Личные/приватные материалы (в .gitignore)
│   ├── MEETING_SCRIPT.md              # Скрипт встречи с руководителем
│   └── Лукашев_Кирилл_отчет_нир_v2.docx.pdf        # Отчёт НИР с ФИО
│
└── .cursor/rules/
    └── thesis-writing.mdc             # Правила для AI при написании глав
```

### Зоны репозитория

- **Корень** — только `README.md` и `STYLE_GUIDE.md` (используются постоянно).
- **`data/`, `notebooks/`** — код и данные исследования; источник истины для всех цифр в ВКР.
- **`thesis/`** — текст диплома по главам.
- **`docs/`** — публичные справочные материалы (методички, ГОСТы, шаблоны).
- **`.local/`** — приватные материалы (отчёты с ФИО, рабочие черновики, скрипты встреч); исключена через `.gitignore`.

### Конвенции именования файлов

- Файлы глав именуются `NN_section_name.md` (двузначный префикс для сортировки).
- Рисунки для диплома хранятся в `data/processed/` (уже используются в README).
- Новые рисунки под диплом — также в `data/processed/`, с описательным именем.

## 14. Быстрая памятка

| Что | Как в ВКР |
| --- | --- |
| Заголовок главы | `1 Название` — жирный, 14pt, с новой страницы |
| Заголовок параграфа | `1.1 Название` — жирный, 12pt |
| Рисунок | `Рисунок 1.2 — Название` — после рисунка, по центру |
| Таблица | `Таблица 1.2 — Название` — над таблицей, по центру |
| Формула | выровнена по центру, номер `(1.2)` справа |
| Ненумерованный список | маркер `–` (тире) |
| Нумерованный список (одно предложение) | `1)` с маленькой буквы, заканчивается `;` |
| Нумерованный список (несколько предложений) | `1.` с большой буквы, заканчивается `.` |
| Ссылка на источник | внутритекстовая `(Автор, год)` или сноска `¹` |
| Лицо повествования | третье: «были получены», «автором показано» |
| Время для своих результатов | прошедшее: «позволило», «показало» |
| Время для описания области | настоящее: «криптовалюты представляют собой» |
