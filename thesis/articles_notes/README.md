# thesis/articles_notes — конспекты научных статей

По одному `.md` на статью. PDF лежат локально в `.local/articles/`,
публичный реестр статей — `docs/articles/README.md`. Цель этих
конспектов — иметь под рукой выжимку для главы 1 (обзор литературы)
без необходимости перечитывать оригинал.

## Текущий статус

Уже заведено **18 заготовок** (по числу статей в реестре). Все они с
готовой шапкой и аннотацией; разделы «Метод», «Результаты», «Цитата
для главы 1», «Связь с нашей работой» остаётся заполнить вручную при
чтении.

## Конвенция именования

Имя файла повторяет ID статьи из реестра (`<Год>_<FirstAuthor>` или
`<Год>_<FirstAuthor>_<SecondAuthor>` для русских коавторов):

```
2018_Liu_Tsyvinski.md         ↔  2018_Liu_Tsyvinski_crypto_risks_returns.pdf
2021_Shilov_Zubarev.md        ↔  2021_Shilov_Zubarev_btc_evolution.pdf
2025_Veloso.md                ↔  2025_Veloso_btc_halving_2024.pdf
```

## Структура конспекта

Каждый файл начинается с YAML-шапки:

```yaml
---
id: 2018_Liu_Tsyvinski
title: "Risks and Returns of Cryptocurrency"
citation_short: "Liu Y., Tsyvinski A. (2018; опубл. 2021)"
group: classics             # классификация из docs/articles/README.md
pdf: "../../.local/articles/2018_Liu_Tsyvinski_crypto_risks_returns.pdf"
core: false                 # true, если статья войдёт в основной список ≥10 штук
---
```

И далее — секции:

– `## Аннотация` — короткая выжимка из метаданных PDF (уже заполнена).
– `## Тип работы` — empirical / theoretical / survey / review.
– `## Вопрос исследования` — 1–2 предложения.
– `## Метод` — econometrics + период данных + выборка.
– `## Ключевые результаты` — 3–5 пунктов.
– `## Цитата для главы 1` — 1–2 предложения готовой цитаты.
– `## Связь с нашей работой` — какую гипотезу/блок поддерживает.

## Как использовать при написании

1. Перед началом главы 1 — пройтись по всем конспектам и выписать
   тематические блоки (внутренние факторы / внешние факторы / гипотезы).
2. При цитировании — брать формулировку из «Цитата для главы 1»
   (она уже выверена), не возвращаясь к PDF.
3. После того как из 18 статей отобраны ≥10 для основного списка,
   проставить `core: true` в их YAML-шапке.
4. Список литературы `thesis/references.md` собирается из тех
   конспектов, у которых `core: true`.
