## 2.1 Данные и переменные

### 2.1.1 Описание выборки

В работе использован набор еженедельных наблюдений за период с 29 января 2017 года по 23 февраля 2025 года (n = 422). Данные разбиты на два подпериода: незрелый рынок (2017–2019, n = 153) и зрелый рынок (2020–2025, n = 269). Граница проведена по 2020 году, поскольку именно с этого периода фиксируется устойчивый рост институционального участия в криптовалютном рынке.

### 2.1.2 Переменные

<!-- Описание каждой переменной: источник, трансформация, обоснование включения -->

**Зависимая переменная:**

- `r_btc` — логарифмическая доходность Bitcoin, рассчитана как $r_t = \ln(P_t / P_{t-1})$, где $P_t$ — цена закрытия пятницы.

**Крипто-специфический блок:**

- `r_btc_lag` — лаг доходности на одну неделю; прокси для momentum.
- `log_volume_btc` — натуральный логарифм объёма торгов Bitcoin за неделю; прокси ликвидности.
- `google_trends` — нормированный индекс Google Trends по запросу «Bitcoin» (0–100); прокси инвесторского внимания.

**Блок внешних рыночных факторов:**

- `r_sp500` — логарифмическая доходность индекса S&P 500.
- `vix` — уровень индекса CBOE VIX; мера глобальной рыночной неопределённости.
- `r_dxy` — логарифмическая доходность индекса доллара США (DXY).

Итоговый датасет находится в `data/processed/merged_weekly.csv`. В датасете отсутствуют пропуски по всем переменным.

## 2.2 Спецификация моделей

Были оценены три OLS-спецификации с одинаковой зависимой переменной `r_btc`:

$$
r_{BTC,t} = \alpha + \sum_{k} \beta_k X_{k,t} + \varepsilon_t \tag{2.1}
$$

где $X_{k,t}$ — набор объясняющих переменных, $\varepsilon_t$ — остаток.

- **Модель М1** (только крипто-специфические факторы): $X = \{r_{btc,lag}, \log{volume}, google\}$
- **Модель М2** (только внешние факторы): $X = \{r_{sp500}, vix, r_{dxy}\}$
- **Модель М3** (полная модель): $X = \{r_{btc,lag}, \log{volume}, google, r_{sp500}, vix, r_{dxy}\}$

## 2.3 Корректировка стандартных ошибок

Поскольку в остатках OLS-моделей были обнаружены ARCH-эффекты (тест Энгла), для корректной статистической инференции применялись HAC-стандартные ошибки (Newey-West). Ширина лага определялась по формуле $h = \lfloor 4(T/100)^{2/9} \rfloor$.

## 2.4 Диагностика

Перед оцениванием выполнены следующие диагностические проверки:

1. **Стационарность:** тест Дики-Фуллера (ADF) — все переменные стационарны.
2. **Мультиколлинеарность:** коэффициент VIF по полной модели находится в диапазоне 1,0–1,3, серьёзной мультиколлинеарности не выявлено.
3. **ARCH-эффекты:** тест Энгла подтвердил их наличие в остатках, что обосновывает применение GARCH(1,1) как проверки устойчивости.

## 2.5 Методы проверки устойчивости

### 2.5.1 GARCH(1,1)

Для проверки устойчивости основных выводов оценена модель GARCH(1,1):

$$
r_{BTC,t} = \mu + \sum_k \beta_k X_{k,t} + \varepsilon_t, \quad \varepsilon_t = \sigma_t z_t \tag{2.2}
$$

$$
\sigma_t^2 = \omega + \alpha \varepsilon_{t-1}^2 + \beta \sigma_{t-1}^2 \tag{2.3}
$$

где $\sigma_t^2$ — условная дисперсия, $z_t \sim N(0,1)$.

### 2.5.2 Rolling window regression

Для анализа временно́й динамики коэффициентов применён метод скользящего окна шириной 52 недели (один год) с шагом одна неделя.

### 2.5.3 Квантильная регрессия

Квантильная регрессия позволяет исследовать связь факторов с доходностью в разных частях её условного распределения. Оцениваются квантили $\tau \in \{0.10, 0.25, 0.50, 0.75, 0.90\}$.
